# SPEC-006: Meeting-Recording Linking & AI Analysis

## Problem Statement
The current meeting-recording linking system in HiDock Next is prone to false positives, especially when candidate selection is limited. The two-stage matching algorithm (Time-based + AI-based) lacks sufficient safeguards to prevent "rubber-stamping" of incorrect matches. Additionally, the AI analysis process generates English content even for non-English transcripts, and the UI fails to correctly prioritize and display AI-generated titles when a meeting is linked.

## Bugs Addressed
- **BUG-009 (P1)**: Wrong meeting linked due to insufficient candidates and lack of attendee validation.
- **BUG-019 (P2)**: AI analysis (summary, action items) defaults to English regardless of transcript language.
- **BUG-020 (P3)**: AI-generated `title_suggestion` is ignored by the UI when a meeting is linked.

## User Stories
- As a user, I want my recordings to be automatically linked to the correct calendar meeting with high confidence.
- As a user, I want to be prompted to confirm a meeting link if the AI is not certain, rather than having a wrong link forced.
- As a Spanish-speaking user, I want my meeting summaries and action items to be in Spanish if the meeting was held in Spanish.
- As a user, I want to see the descriptive title generated by AI, even if the recording is linked to a meeting with a generic subject.

## Current Behavior (Broken)
1. **Candidate Selection**: `findCandidateMeetingsForRecording` uses a strict ±30min window. If only one meeting is found, the AI prompt effectively forces a match.
2. **AI Matching**: Gemini is given a list of candidates and asked to pick one. If only 1 candidate exists, Gemini often "confirms" it even if the content doesn't match, because the prompt says "This recording appears to be from the meeting... Verify this makes sense."
3. **Language Handling**: The analysis prompt is 100% English. While it asks for a `"language"` field, it doesn't instruct the model to use that language for the summary or action items.
4. **Title Display**: `SourceReader.tsx` prioritizes `meeting.subject` over `recording.title`. Since `updateKnowledgeCaptureTitle` saves the AI suggestion to `recording.title`, it is hidden by the meeting subject.

## Expected Behavior (Target)
1. **Smart Candidates**: Expand candidate search and pre-sync calendar if needed. Include attendee names in the context for AI to verify against the transcript.
2. **Threshold-based Linking**: 
   - High Confidence (>0.85) + Multi-signal match (Time + Attendees/Topics) -> Auto-link.
   - Low Confidence (<0.60) -> No link, just suggest.
   - Middle Ground -> Link but flag as "Pending Confirmation".
3. **Multilingual Analysis**: The system detects the transcript language and ensures the AI response (summary, title, etc.) is in that same language.
4. **Unified Title UX**: Display the AI-generated descriptive title as the primary heading, and show the linked meeting subject as a secondary "location/context" badge.

## Acceptance Criteria
- [ ] AI matching must reject a single candidate if the transcript content has zero relevance to the meeting subject/description.
- [ ] Attendee names from the calendar must be used as a secondary signal for matching.
- [ ] AI summaries and action items must be generated in the same language as the transcript (verified for English and Spanish).
- [ ] `title_suggestion` must be visible in the Library and Reader even when a meeting is linked.
- [ ] Users must be able to "Unlink" or "Correct" a meeting link (requires SPEC-002).

## Technical Approach

### Improved Candidate Selection
- Expand search window to ±60 minutes to capture meetings that started late or recordings that started very early.
- Include `attendees` (names/emails) in the `Meeting` object passed to Gemini.
- **Pre-sync Trigger**: Check if calendar sync has run in the last 15 minutes before starting transcription matching.

### AI Matching Safeguards
- **Modified Prompt**: Instead of "Verify this makes sense" for single candidates, use:
  > "If there is only one candidate, evaluate if it is a GENUINE match. If the transcript content (topics, participants) contradicts the meeting details, return 'null' for selected_meeting_id and a confidence of 0.0."
- **Attendee Verification**: Add a specific instruction to check if names mentioned in the transcript match the meeting attendee list.
- **Confidence Thresholds**:
  - `CONFIDENCE_AUTO_LINK = 0.85`
  - `CONFIDENCE_SUGGEST_ONLY = 0.50`

### Language-Aware Analysis Prompt
Implement a two-step prompt or a clearer single prompt:
```typescript
const analysisPrompt = `
First, identify the primary language of the transcript below (e.g., "en", "es").
Then, perform the following analysis. 
CRITICAL: All text fields in your JSON response (summary, title_suggestion, action_items, key_points, selection_reason) MUST be written in the SAME language as the transcript.
... rest of prompt ...
`;
```

### Title Display Logic
Update `SourceReader.tsx` and `SourceRow.tsx`:
- **Primary Title**: `recording.title || meeting?.subject || recording.filename`
- **Context Badge**: If `meeting` exists, show `📅 {meeting.subject}` below the title or as a badge.
- This ensures the AI's "Daily Standup - Discussing API migration" (descriptive) is seen over "Internal Daily" (generic meeting subject).

### Manual Override UX
- Add an "Unlink" button in the `SourceReader` metadata section.
- Add a "Change Meeting" button that opens the `LinkMeetingDialog`.
- *Note: These rely on the `recording_id` stability being addressed in SPEC-002.*

## UI/UX Requirements
- **Library View**: Show both the descriptive title and the meeting badge.
- **Reader Header**:
  - Row 1: `title_suggestion` (Large)
  - Row 2: Meeting Subject (Icon + Link style), Date/Time
- **Confidence Indicator**: Show a subtle "AI Matched" vs "User Linked" badge.

## Testing Strategy
- **Test Case 1 (Cross-match)**: Record a tech interview during a "Marketing Sync" time slot. AI should reject the link.
- **Test Case 2 (Spanish)**: Process a Spanish transcript. Verify summary and action items are in Spanish.
- **Test Case 3 (Attendee Signal)**: Two overlapping meetings. Transcript mentions "Sebastian" (attendee of Meeting A). AI should select Meeting A.
- **Test Case 4 (Title Display)**: Link a recording to a meeting. Verify `title_suggestion` is displayed in the UI instead of being hidden by the meeting subject.

## Dependencies & Risks
- **Dependency**: SPEC-002 (Recording ID stability). If recording IDs change on sync, manual overrides will be lost.
- **Risk**: Gemini might still hallucinate matches if the "No Match" option isn't emphasized enough.

## Out of Scope
- Automatic creation of calendar events from recordings.
- Supporting more than 2 languages (EN/ES focus for now).
- Real-time matching during recording (post-processing only).
